# Future Module: A3 Мастер

Статус: идея сохранена в roadmap, не в активной разработке.

## Важно

Этот модуль нельзя внедрять в текущий этап A3 Condition Engine, Visual Workflow Builder и A3 Automation Platform без отдельной задачи.

Сейчас не реализовывать:

- голосовое распознавание
- AI-command execution
- новый тариф
- новый UI
- мобильный режим
- оплату тарифа
- изменения текущих ролей
- изменения текущего A3 engine

Цель этого документа: сохранить идею и архитектурное направление, чтобы позже вернуться к модулю отдельным этапом.

## Идея

`A3 Мастер` — отдельный простой режим платформы для частных специалистов, которые работают сами на себя:

- монтажники кондиционеров
- холодильщики
- сантехники
- электрики
- мастера по ремонту
- клинеры
- мастера бытовой техники
- мебельщики
- частные специалисты

Главный принцип: мастер не заполняет CRM руками. Мастер говорит голосом, а AI-помощник сам создаёт клиентов, заявки, напоминания и записи в календаре.

## Отличие От Основной A3 Platform

Основная A3 Platform:

- A3 workflow
- SLA

## Главный Экран

На главном экране должно быть максимум пять основных действий:

1. Сегодня
2. Новая запись
3. Голос
4. Клиенты
5. Деньги

В центре желательно большая кнопка микрофона.

## Голосовые Команды

Создание записи:

```text
Запиши Сергея на завтра в 15:00. Установка кондиционера. Адрес Ленина 25.
```

AI должен распознать имя, дату, время, услугу, адрес, телефон, создать клиента при необходимости, создать заявку, поставить её в календарь и добавить напоминание.

Перенос записи:

```text
Перенеси Иванова с пятницы на субботу на 10 утра.
```

AI должен найти клиента или заявку, изменить дату и время, обновить календарь и сохранить историю.

Поиск клиента:

```text
Найди клиента Сергея с кондиционером Mitsubishi.
```

AI должен искать по имени, телефону, адресу, описанию заявки, модели оборудования и истории работ.

Завершение работы:

```text
Работу закончил. Клиент заплатил 18 тысяч переводом. Заменил компрессор. Гарантия год.
```

AI должен закрыть заявку, отметить оплату, записать сумму, сохранить отчёт, добавить гарантию и обновить историю клиента.

Напоминания:

```text
Напомни мне позвонить Сергею через два дня.
```

AI должен создать напоминание, привязать его к клиенту, если клиент понятен, и показать в календаре.

Обслуживание:

```text
Напомни через полгода предложить обслуживание кондиционера.
```

AI должен создать будущую задачу, связать её с клиентом и поставить напоминание.

Деньги:

```text
Сегодня получил 15 тысяч наличными.
```

AI должен добавить доход, связать его с заявкой, если понятно, и обновить дневную выручку.

Расписание:

```text
Кто у меня завтра?
```

AI должен показать заявки на завтра: время, клиента, адрес и услугу.

## Рабочий День

Будущий режим: `Начать рабочий день`.

После запуска система показывает:

- ближайшего клиента
- маршрут
- время
- адрес
- историю клиента
- заметки
- что нужно сделать

Будущий режим: `Закончить рабочий день`.

После завершения система показывает:

- сколько заявок выполнено
- сколько денег получено
- кто не оплатил
- кому нужно перезвонить
- что запланировано на завтра

## Будущий Тариф

Рабочее название тарифа: `Мастер`.

Ориентир цены: `500-1000 ₽ / месяц`.

В тарифе не должно быть:

- сложной аналитики компании
- ролей `boss`, `manager`, `worker`
- управления сотрудниками
- корпоративных workflow
- сложного A3 governance
- тяжёлых dashboards

В тарифе должно быть:

- клиенты
- записи
- календарь
- напоминания
- голосовой помощник
- простой доход
- история работ
- фото до/после
- гарантия
- простые автоматические сообщения

## Архитектурный Принцип

Не делать отдельное приложение с нуля.

Лучше сделать отдельный режим внутри текущей платформы:

```text
mode = master
```

или:

```text
plan = master
```

Один backend, одна база, одна архитектура, но другой UI и другой набор функций.

Модуль должен использовать существующие сущности:

- `clients`
- `tasks`
- `calendar`
- `reminders`
- `task_files`
- `finance`
- `automation`

UI должен быть максимально упрощённым.

## Когда Вернуться

Вернуться к этому модулю отдельным этапом после стабилизации:

- A3 Condition Engine

- A3 Automation Platform
- governance safety
- базового production hardening
